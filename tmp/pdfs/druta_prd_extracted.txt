1. Executive Summary
Druta is a real-time competitive running platform that transforms physical running into a global,
multiplayer experience. The app combines GPS-based run tracking, territory claiming
mechanics, and live head-to-head racing between friends across the world.
The core idea is simple: running should feel competitive, social, and strategic, not solitary and
repetitive.
Users can:
● Claim and defend map territories by running in the real world.
● Challenge friends globally to live races.
● See real-time race progress on a shared virtual track.
● Compete for rankings, wins, and territory dominance.
The product sits at the intersection of fitness, gaming, and social competition.
2. Problem Statement
2.1 Core Problem

Most users quit running apps within months because:
● Running feels lonely.

4. Core Product Features
4.1 Territory Claiming System

===== PAGE 3 =====
The world map is divided into 200m x 200m grid squares.
When a user runs through a grid:
● The grid becomes claimed.
● Ownership is assigned to the user.
● Repeated runs increase territory strength.
If another user runs a longer distance within the same grid:
● Ownership transfers.
● The previous owner loses control.
Objectives:
● Encourage consistent running.
● Create local competition.
● Introduce map-based gamification.
● Add loss aversion psychology.
The territory layer creates a long-term engagement loop.
4.2 Live Global 1v1 Race Mode
Two users in different cities or countries can race in real time.
Example:
● User A in Mumbai.
● User B in San Francisco.

===== PAGE 4 =====
They initiate a race (distance-based or timed).
During the race:
● Both users’ GPS coordinates are tracked.
● The server calculates authoritative distance.
● The app displays both avatars on a shared virtual track.
● Distance and gap update in real time.
● The winner is determined server-side.
Race types:
1. Distance Race (e.g., first to 5 km wins)
2. Timed Race (e.g., whoever runs farther in 30 minutes wins)
This creates synchronous competition across geographies.
4.3 Group Territory Conquest
Users can form running groups (teams/clans) to collectively dominate territories within a city.
Groups may represent:
● Friends
● Running clubs
● University teams
● Corporate teams
● Local communities
When group members run:

===== PAGE 5 =====
● Their runs contribute to group territory strength.
● Territories can display both individual and group ownership layers.
● Groups can capture large areas of a city collectively.
If rival groups run more distance within a territory:
● Control can transfer between groups.
Objectives
● Encourage social running and community building.
● Create city-level rivalry between groups.
● Increase daily engagement through cooperative gameplay.
● Enable team-based strategy in territory control.
● Strengthen retention through social identity and group competition.
